🔄 Como Funciona o Redirecionamento

1

Usuário clica em link antigo

Exemplo do banco de dados Django:

https://curriculo.sedu.es.gov.br/curriculo/wp-content/uploads/2023/09/arquivo.pdf
2

Apache detecta `/wp-content/`

O servidor lê a regra de reescrita no `.htaccess`:

RewriteRule ^curriculo/wp-content/(.*)$ \ https://wordpress.curriculo.sedu.es.gov.br/curriculo/wp-content/$1 [R=301]
3

Redirecionamento HTTP 301

Servidor responde com "Moved Permanently" para o subdomínio WordPress

HTTP/1.1 301 Moved Permanently Location: https://wordpress.curriculo.sedu.es.gov.br/curriculo/wp-content/uploads/2023/09/arquivo.pdf
4

Navegador segue automaticamente

Arquivo é entregue do WordPress em subdomínio com status 200 OK

HTTP/1.1 200 OK Content-Type: application/pdf Content-Length: 1234567

🎯 Resumo Executivo

  • ✅ O que vai funcionar: Todos os links do tipo `/curriculo/wp-content/...` vão continuar funcionando transparentemente
  • ✅ Sem duplicação: Os 1000+ arquivos continuam no WordPress (em subdomínio), não precisam ser copiados
  • ✅ Sem alterações no banco: Nenhum link no Django precisa ser modificado — a reescrita é automática
  • ✅ Transparente ao usuário: Quem clicar em um arquivo vai vê-lo abrir/baixar normalmente, como se estivesse no mesmo lugar
  • ✅ Fácil de remover depois: Quando todos os arquivos forem transferidos, o subdomínio WordPress pode ser deletado

🔗 Ligação técnica: Apache RewriteRule (no `.htaccess`) + Apache RewriteEngine (redirecionamento 301 permanente) = redirecionamento automático e indexável pelos buscadores
